Conditions on text

Conditions can be applied to text . This allows the same content to be published with slight modifications for different audiences.

Conditions save you writing effort.

Conditions allow authors to save time by writing a given topic in a way that includes several different variants of the subject. For example, a topic describing a software API can include conditions specifying the version at which a given API was introduced. The documentation may then be published for all versions of the API from a single source.

A condition has a name and a value. For example, for the conditionÂ audience beginner, the condition name is audience and the value is beginner. The set of condition names is fixed and will be presented in a combo box when you create a new condition. Condition values are free form text strings that cannot include spaces. You should agree with your colleagues on what values should be used for different condition names.

Conditions Bars

Conditions are indicated in AuthorBridge with colored bars. Overline bars are used when conditions are applied to short phrases. Vertical bars are shown in the right margin for conditions that apply to block structures such as paragraphs, lists, notes, and tables.

Each color represents a unique combination of conditions applied at a single point.



Conditions Sidebar

The conditions sidebar is available from the Toolbox. It is divided into three parts:

  • The first part displays a summary of all applied conditions at the cursor position.

  • The second part, which is expandable, is used for adding and removing conditions.

  • The third part presents the color legend and allows you to filter the view of the topic.
