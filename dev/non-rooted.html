Non-rooted topics

A non-rooted topic is one that does not have a home. It has no associated storage location in the repository.

How non-rooted topics are created

When you create a new topic in AuthorBridge or open one from a local , it is non-rooted because AuthorBridge does not know where you intend for it to reside. It is temporarily residing right inside the browser.

Drawbacks of non-rooted topics

When topics are non-rooted, you can create most of your content but you cannot do any of the following:
  • create or paste links, and
  • view or browse for an image.

You have limited functionality for adding a graphic. You can add it by file name only.

How to fix the problem

We recommend integrating repository for storing topics. You root a topic by saving it to the configured repository. Unfortunately, you currently have to re-open it from the repository to then have the rooted-version.

Contact Stilo for more information on your integration and repository options.
