Use sections appropriately

Put the right kind of information in each section.

One of the advantages of using AuthorBridge over a word processing tool like Microsoft Word is that you are using sections to indicate the type of content included therein. This is called semantic markup.

Semantic markup gives the content incredible versatility, especially during the publishing process. Conversely, content that is in the wrong section (a section that doesn't match the actual content) can often become a problem during publishing.

For example, you put things that a user needs to do before they begin a task in a section called Before you begin rather than any other section. Inversely, you would never put any other kind of information in the Before you begin section, even if it looks like a good location for it.

Your publishing process might automatically add the word Prerequisite before the text of any Before you begin section. If you have content that is not a prerequisite in Before you begin, then the result confuses readers.

  1. As you write your topics, use the hover help for how to use the different sections.
    The topic types themselves define the available sections to some degree. For example, you can only put steps in a task topic because that's where all procedural information should go.
  2. Review your content for appropriate section use.
    1. Where the content doesn't match the purpose of the section, move it to an appropriate section.
    2. Use the functionality on the left to add and remove sections.
  3. Get a peer review.
    Often another writer can quickly see if you have put information in the wrong section.
  4. Update the content and sections, as needed.
    In some instances, you re-write content to fit the section you are using; more often, you should create the type of section you need and move the content into it.
